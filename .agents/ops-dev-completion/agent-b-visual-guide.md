# Agent B - 파생 상태 시각 가이드

## 파생 상태 표시 예시

### Inspector 패널

```
┌─────────────────────────────────────────┐
│ Agent Inspector                          │
├─────────────────────────────────────────┤
│  🔴  Agent Alpha                         │
│      개발 · 메인 에이전트                 │
│                        [작업 중]          │
│                        [오류 발생]  ← 파생 │
└─────────────────────────────────────────┘
```

**설명**: 기본 상태 "작업 중" 아래에 파생 상태 "오류 발생" 배지가 추가 표시됩니다.

- **테두리**: 연한 빨간색

#### 3. 차단 상태 (blocked)
```
┌─────────────────────────────────────────┐
│ 🔴💥 Agent Alpha                   휴식  │
│    개발 · my-project · 차단됨             │
└─────────────────────────────────────────┘
```
- **dot**: 진한 빨간색 + 강한 빛남 + 펄스 애니메이션
- **테두리**: 진한 빨간색
- **배경**: 연한 빨간색
- **텍스트**: "차단됨" 추가

#### 4. 활발한 상태 (recently_active)
```
┌─────────────────────────────────────────┐
│ 🟢✨ Agent Alpha                  작업 중 │
│    개발 · my-project · 활발함             │
└─────────────────────────────────────────┘
```
- **dot**: 청록색 + 강한 빛남 + 펄스 애니메이션
- **테두리**: 연한 청록색
- **텍스트**: "활발함" 추가

## 상태 추론 로직 플로우

```
Agent 상태 확인
    ↓
recentActivity 있나?
    ↓ YES
마지막 활동 = tool_result?
    ↓ YES
is_error = true?
    ↓ YES
현재 status = idle?
    ↓ YES → BLOCKED
    ↓ NO  → ERROR

마지막 활동 5초 이내?
    ↓ YES
현재 status = working?
    ↓ YES → RECENTLY_ACTIVE
    ↓ NO  → null
```

## 색상 체계

### error (오류 발생)
- **배지 배경**: `rgba(240, 103, 103, 0.18)`
- **배지 텍스트**: `#ff9b9b`
- **dot**: `#f06767` + `box-shadow: 0 0 12px rgba(240, 103, 103, 0.6)`

- **row 배경**: `rgba(240, 103, 103, 0.08)`

- **배지 텍스트**: `#7fdccf`

## 사용 시나리오

### 시나리오 1: 도구 실행 실패
1. Agent가 `Bash` 도구로 명령어 실행
2. 명령어 실패 (exit code != 0)
3. `tool_result`에 `is_error: true` 기록
4. UI에 **error** 파생 상태 표시
5. 사용자가 즉시 문제 인지

### 시나리오 2: 에러 후 정지
1. Agent가 도구 실행 중 에러 발생
2. Agent 상태가 `idle`로 전환
3. UI에 **blocked** 파생 상태 표시
4. 펄스 애니메이션으로 주의 환기
5. 사용자 개입 필요

### 시나리오 3: 활발한 작업
1. Agent가 연속적으로 도구 사용
2. 최근 5초 이내 활동 + `working` 상태
3. UI에 **recently_active** 파생 상태 표시
4. 긍정적인 청록색 펄스로 활발함 강조

## 테스트 방법

### 에러 상태 테스트
```bash
# 의도적으로 실패하는 명령어 실행
claude code "Run bash command: false"
```
**기대 결과**: error 파생 상태 표시

### 차단 상태 테스트
```bash
# 에러 후 대기
claude code "Run bash command: exit 1"
# 에이전트가 idle로 전환될 때까지 대기
```
**기대 결과**: blocked 파생 상태 + 펄스 애니메이션

### 활발한 상태 테스트
```bash
# 연속 작업 요청
claude code "Read all .ts files and summarize them"
```
**기대 결과**: recently_active 파생 상태 + 펄스 애니메이션

## 향후 개선

### approval_needed 상태 활성화
현재는 주석 처리되어 있습니다. 서버에서 `lastActivity` 타임스탬프를 제공하면 다음과 같이 활성화할 수 있습니다:

```typescript
// 장시간 waiting 확인 (30초 이상)
if (agent.status === 'waiting' && agent.lastActivity) {
  const waitingTime = (now - new Date(agent.lastActivity).getTime()) / 1000
  if (waitingTime > 30) {
    return 'approval_needed'
  }
}
```

**표시 예시**:
```
┌─────────────────────────────────────────┐
│ 🟠 Agent Alpha                    대기   │
│    개발 · my-project · 승인 대기          │
└─────────────────────────────────────────┘
```

## 디자인 원칙

1. **비침습적**: 기본 상태를 가리지 않음
2. **명확성**: 파생 상태는 작은 폰트/연한 색상으로 "추정"임을 표현
3. **즉각성**: 중요한 상태는 애니메이션으로 주의 환기
4. **일관성**: 모든 파생 상태는 동일한 UI 패턴 사용
