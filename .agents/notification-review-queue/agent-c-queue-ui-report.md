## UI 구조 설명

### 1. 컴포넌트 위치
- **위치**: Staff Board 섹션 최상단
- **배치**: Staff List 위에 독립적인 섹션으로 표시
- **조건부 렌더링**: 검수 대기 중인 에이전트가 있을 때만 표시

### 2. 검수 큐 헤더
```
┌─────────────────────────────────────┐
│ 검수 큐              [3] [2] [1]    │
│ 총 6개 검수 대기                    │
└─────────────────────────────────────┘
```

- **제목**: "검수 큐"
- **요약**: 총 검수 대기 개수
- **상태별 배지**:
  - 강한 초록 배지: pending 상태 개수
  - 연한 초록 배지: acknowledged/copied 상태 개수
  - 회색 배지: dismissed 상태 개수

### 3. 검수 큐 리스트

각 항목은 다음 정보를 표시:
- **상태 도트**: 시각적 상태 표시
  - pending: 강한 초록 (애니메이션)
  - acknowledged/copied: 연한 초록
  - dismissed: 회색
- **에이전트명**: 클릭 가능
- **소스 배지**: Claude(C) 또는 Codex(X)
- **대기 시간**: "5분 전", "2시간 전" 등
- **상태 라벨**: "검수 대기", "확인함", "복사 완료", "숨김"

### 4. 정렬 방식
- **기본 정렬**: 시간순 (오래된 것이 위로)
- **근거**: 가장 오래 대기한 검수 요청을 우선 처리

### 5. 클릭 동작
- 검수 큐 항목 클릭 시 해당 에이전트가 Inspector에 자동 선택됨
- 기존 Staff List와 동일한 상호작용 패턴 유지

## 스타일링 특징

### 1. 상태별 색상 구분
- **pending**:
  - 배경: `rgba(48, 195, 168, 0.08)`
  - 테두리: `rgba(48, 195, 168, 0.3)`
  - 강한 시각적 강조 (펄스 애니메이션)

- **acknowledged/copied**:
  - 배경: `rgba(48, 195, 168, 0.04)`
  - 테두리: `rgba(48, 195, 168, 0.15)`
  - 불투명도: 0.85 (약간 흐림)

- **dismissed**:
  - 배경: `rgba(255, 255, 255, 0.02)`
  - 테두리: `rgba(255, 255, 255, 0.04)`
  - 불투명도: 0.5 (매우 흐림)

### 2. 반응형 디자인
- **데스크톱**: 검수 큐 최대 높이 240px, 스크롤 가능
- **모바일** (720px 이하):
  - 최대 높이 200px로 축소
  - 폰트 크기 및 패딩 조정
  - 터치 친화적 버튼 크기 유지

### 3. 레이아웃 호환성
- 기존 3컬럼 레이아웃 유지
- Staff Board 내부 섹션으로 추가되어 별도 공간 차지 없음
- iPad/데스크톱에서 텍스트 겹침 없이 깔끔하게 표시

## 기능 상세

### 1. 필터링
현재 구현:
- 검수가 필요한 에이전트만 자동 필터링 (`needsReview === true`)
- 검수가 없으면 컴포넌트 자체를 숨김

향후 확장 가능:
- 상태별 필터 (pending만 보기 등)
- 프로젝트별 필터
- 소스별 필터 (Claude/Codex)

### 2. 개수 표시
- **총 검수 필요 개수**: 헤더 요약에 표시
- **상태별 개수**:
  - `pendingCount`: pending 상태
  - `acknowledgedCount + copiedCount`: 처리된 상태
  - `dismissedCount`: 숨긴 상태

### 3. 시간 표시
- `formatTimeSince()` 함수 사용
- 상대적 시간 표시:
  - 60초 미만: "방금"
  - 60분 미만: "N분"
  - 24시간 미만: "N시간"
  - 24시간 이상: "N일"

## 기술적 구현

### Props 인터페이스
```typescript
interface ReviewQueueProps {
  agents: Agent[]
  onSelectAgent: (agentId: string) => void
}
```

### 주요 로직
1. **필터링**: `agents.filter(agent => agent.needsReview)`
2. **정렬**: `reviewCandidateAt` 타임스탬프 기준 오름차순
3. **상태 집계**: 각 reviewState별 개수 계산
4. **조건부 렌더링**: 검수 대기 중인 에이전트가 없으면 null 반환

## 테스트 시나리오

### 1. 검수 큐 없을 때
- 검수 필요 에이전트 없음
- ReviewQueue 컴포넌트가 렌더링되지 않음
- Staff List만 표시됨

### 2. 검수 큐 있을 때
- 검수 필요 에이전트 존재
- ReviewQueue가 Staff List 위에 표시
- 상태별 개수 배지 정확히 표시

### 3. 항목 클릭
- 검수 큐 항목 클릭
- Inspector에 해당 에이전트 자동 선택
- Review Banner 표시됨

### 4. 상태 변경
- Inspector에서 상태 변경 (acknowledged, copied, dismissed)
- 검수 큐 항목의 스타일 자동 업데이트
- 개수 배지 실시간 업데이트

### 5. 반응형
- 모바일에서 레이아웃 깨짐 없음
- 터치 인터랙션 정상 작동
- 스크롤 영역 적절히 조정

## 결론

검수 큐 UI가 성공적으로 구현되었으며, 다음과 같은 목표를 달성했습니다:

1. ✅ Staff Board 내부에 깔끔하게 통합
2. ✅ 상태별 색상 구분으로 시각적 우선순위 제공
3. ✅ 시간순 정렬로 긴급도 파악 용이
4. ✅ 클릭 시 에이전트 선택으로 빠른 대응 가능
5. ✅ 반응형 디자인으로 모든 화면 크기 지원
6. ✅ 개수 표시로 전체 검수 현황 한눈에 파악

기존 Staff Board 기능을 해치지 않으면서도 검수 관리 기능을 효과적으로 추가했습니다.
